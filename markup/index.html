Query2JSON

What is Query2JSON?

Query2JSON solves a very simple and (arguably) uncommon problem. Several services (i.e. Twilio) use webhooks to allow applications to react to incoming data. These services make requests like this when something happens (for example, an SMS is received).
[Event Happens on FirstService]
GET http://www.yourwebsite.com/callback?cat=hat&fish=stick
This works fine if you're writing a big 'ole custom application with servers and whatnot, but sometimes you need to "glue" the data from this service into another service or database host (i.e. Firebase) without all that hassle. However, not all services like to talk in exactly the same way and thus cannot talk directly to each other. Firebase, for example, uses a JSON blob method to write data where the JSON is uploaded as the body of the request like this.
http://www.secondservice.com/youraccount/
-> {"cat": "hat", "fish":"stick"}
Sometimes you're lucky and the services you want to use are inheritly capable of sharing data and sometimes you're not. The problem arises when a service like Twilio (which uses HTTP query data method) attmepts to talk to a service like Firebase (which uses the JSON blob method). These services are extremely close to working together, but not quite. Query2JSON solves this little incompatibility by providing a way to translate the request without running your own server. A request with Query2JSON works like this.
[Event Happens on FirstService]
http://www.query2json.com/{ID}?cat=hat&fish=stick
[Query2JSON Translates Request]
http://www.secondservice.com/youraccount/
-> {"cat": "hat", "fish":"stick"}
It is as simple as that. You send me query data and I'll make another request with that data transformed into JSON. Boom, done.
